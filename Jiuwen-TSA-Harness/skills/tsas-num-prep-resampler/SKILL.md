# tsas-num-prep-resampler — 时间序列数据时间轴治理

## 1. 职责

这是一个独立的时间序列数据重采样工具。给定**一份已治理的 CSV**（含时间列）和**一份元信息文件**，执行完整的时间轴治理流程：

1. **名义采样间隔推断** — 对时间戳差值取众数
2. **目标网格选择** — 用户指定或自动选择（口径B：修改数据点数最少，平局选粗）
3. **重采样** — 各列自适应策略（下采样取历史最近值 / 上采样前向填充 / 标注列最近邻）
4. **偶发/区间缺失判定** — 连续缺失 ≥ K 为区间缺失
5. **偶发缺失填充** — 业务列前向填充，标注列最近邻，不跨越区间缺失

输出：
- `resampled_data.csv` — 标准等间隔网格上的数据
- `resampled_data_report.md` — 重采样过程报告（间隔选择、各列方法、缺失统计）

## 2. 输入要求

### 2.1 数据文件

- **已治理的 CSV**，首列为时间列（列名 `time`），其余列为业务数据列或标注列
- 格式：UTF-8，第一行为列头
- 支持 CSV 中的缺失值表现为空字符串或常见 NA 标记

### 2.2 元信息文件（必选）

路径通过 `--metadata` 参数传入，Markdown 格式。详细格式规范见 `references/metadata_spec.md`。

**必须字段：**

| 字段 | 是否必须 | 用途 |
|---|---|---|
| `## 列清单` | **必须** | 声明 CSV 中各列的列名、角色（业务/标注/时间）、类型 |

**可选字段：**

| 字段 | 用途 | 缺失行为 |
|---|---|---|
| `## 标注列` | 显式列出标注列 | 仅以列清单中角色为"标注"的列为准 |
| `## 目标网格间隔` | 指定目标网格秒数 | 自动按口径B选择 |
| `## 区间缺失阈值` | 指定区间缺失判定 K 值 | 默认 10 |

### 2.3 用户参数

CLI 参数 `--grid-interval` 和 `--k` 的优先级高于元信息文件中对应的字段。

## 3. 输出

| 输出项 | 文件名（可定制） | 说明 |
|---|---|---|
| 重采样数据 | `resampled_data.csv` | UTF-8-BOM 编码，标准等间隔网格 |
| 重采样报告 | `resampled_data_report.md` | 间隔选择、各列方法、缺失统计 |

退出码：
- **0**：成功
- **1**：IO/参数错误（文件不存在、格式错误等）
- **2**：数据错误（无法重采样，如缺少时间列）

## 4. 用法

```bash
python resampler.py <input_csv> \
    --metadata <metadata.md> \
    [--output-dir <dir>] \
    [--output <csv文件名>] \
    [--report-output <报告文件名>] \
    [--grid-interval <秒>] \
    [--k <连续缺失阈值>]
```

- `<input_csv>`：已治理的 CSV 文件路径
- `--metadata` / `-m`：元信息文件路径（必需）
- `--output-dir` / `-o`：输出目录（缺省为输入文件所在目录）
- `--output`：CSV 输出文件名（缺省 `resampled_data.csv`）
- `--report-output`：报告输出文件名（缺省 `resampled_data_report.md`）
- `--grid-interval`：目标网格间隔（秒），覆盖自动选择和元信息
- `--k`：区间缺失判定阈值（缺省 10，覆盖元信息）

**Python 环境**：任一 Python 3.11+（含 pandas 和 numpy）。脚本启动时会自检依赖，缺失时打印友好提示。

## 5. 处理流程

### 5.1 解析元信息

从元信息 MD 中提取：
- 列清单（列名 → 角色、类型）
- 标注列名列表
- 可选的目标网格间隔和 K 值

### 5.2 名义采样间隔推断

对全部时间戳的相邻差值取众数，作为数据的"主流"采样频率。

### 5.3 目标网格选择

- 如果用户指定了网格间隔（通过元信息或 CLI），直接使用
- 否则按**口径B**自动选择：对各列（非空值）的众数间隔收集候选池，逐列计算需新增/删除的数据点数，选总修改数据点数最少的间隔；平局选粗网格

### 5.4 重采样

对每列独立执行：
- 下采样（原始间隔 < 目标间隔）：取历史上最近值
- 上采样·业务列：前向填充
- 上采样·标注列：双向最近邻，等距取前值
- 间隔相同：网格对齐

### 5.5 偶发/区间缺失判定

在标准网格上逐列扫描连续缺失段：
- 连续缺失 ≥ K → 区间缺失（标记但不填充）
- 连续缺失 < K → 偶发缺失

### 5.6 偶发缺失填充

- 业务列：前向填充（遇到区间缺失边界停止）
- 标注列：最近邻（双向，等距取前，不跨区间缺失）
- 区间缺失：保持空值

## 6. 报错条件

| 条件 | 退出码 | 提示 |
|---|---|---|
| 元信息文件不存在或不可读 | 1 | 检查路径 |
| CSV 文件不可读 | 1 | 检查路径 |
| 元信息缺少 `## 列清单` | 2 | 元信息须包含列清单 |
| CSV 中没有 `time` 列 | 2 | 数据文件首列须为时间 |
| 数据行数不足以推断间隔 | 2 | 数据点过少 |
| 依赖缺失（pandas/numpy） | 1 | 换用已配置的 Python 环境 |
