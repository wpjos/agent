# tsas-num-prep-resampler 算法规格

> 本文档定义 resampler 的全部处理规则和算法逻辑，供 SKILL.md 引用。

---

## 1. 处理流程总览

```
读 CSV + 读元信息
     ↓
[1] 名义采样间隔推断（众数）
     ↓
[2] 目标网格选择（用户指定 / 口径B自动）
     ↓
[3] 重采样（按列自适应：下采样取历史最近值 / 上采样前向填充 / 标注列最近邻）
     ↓
[4] 偶发/区间缺失判定（K 阈值）
     ↓
[5] 偶发缺失填充（不跨越区间缺失）
     ↓
[6] 输出 CSV + 报告
```

## 2. 算法细节

### 2.1 名义采样间隔推断

对全部时间戳序列的相邻差值取**众数**（mode）。

```
diffs = times.diff().dropna()
nominal_interval = diffs.mode().iloc[0]
```

- 众数为名义间隔，代表数据的"主流"采样频率
- 数据点不足 2 个时，默认 1 秒

### 2.2 目标网格选择

#### 2.2.1 用户指定间隔
若用户通过元信息（`## 目标网格间隔`）或 CLI（`--grid-interval`）传入，直接使用，跳过自动选择。

#### 2.2.2 自动选择（口径B）

候选间隔池 = **各列（非空值）的众数间隔集合**。

> ⚠️ **重要**：不使用合并后全局时间轴的相邻差值。原因：合并后时间轴是所有源文件时间点的并集，高频列（如 1s）会把低频列的时间点"淹没"，导致 3s/5s 等真实采样率在全局差值中消失。此外，缺失点会在高频列中产生伪影间隔（如 1s 采样缺一行 → 出现 2s 差值），这些都不是真实的采样率。
>
> 正确做法：逐列提取非空值对应的时间戳，计算该列的众数间隔，汇总所有列的众数间隔作为候选池。

对每个候选间隔 `cand_td`，逐列计算修改数据点数：

```
grid_set     = 候选间隔生成的等间隔网格时间戳集合
col_ts_set   = 该列所有非空值对应的时间戳集合

aligned   = |col_ts_set ∩ grid_set|      # 精确对齐，无需修改
to_add    = |grid_set| - aligned          # 网格上有但该列无值（上采样需填充）
to_delete = |col_ts_set| - aligned        # 该列有值但网格上无（下采样需丢弃）
col_mods  = to_add + to_delete

总修改数据点数 = Σ col_mods （所有非时间列求和）
```

> ⚠️ **关键**：必须在**逐列单元格级别**计算，而非全局时间轴级别。
> 如果只对合并后并集时间轴做集合差，高频列会"掩盖"低频列的大量上采样需求，
> 导致修改数据点数被严重低估（甚至数量级误差）。

**选择规则**：
```
按总修改数据点数升序排序
平局（修改数据点数相同）→ 选择间隔更大的（粗网格优先）
```

排序键：`(total_modifications, -total_seconds)`

#### 2.2.3 网格生成

从 `time_min` 到 `time_max`，按目标间隔逐步累加（左闭右闭）：

```python
current = time_min
while current <= time_max:
    grid.append(current)
    current += interval


### 2.3 重采样

对每列独立执行重采样。首先计算该列**原始数据的众数间隔**（排除空值后），与目标间隔比较：

| 条件 | 策略 |
|---|---|
| 原始间隔 < 目标间隔 | **下采样** |
| 原始间隔 > 目标间隔 | **上采样** |
| 原始间隔 == 目标间隔 | **网格对齐** |

#### 2.3.1 下采样：取历史最近值

对每个网格时间点 `t`，取原始数据中 `≤ t` 的最大时间点对应的值。
若 `t` 早于所有原始数据，取第一个原始值。

```python
pos = bisect_right(orig_times, t) - 1
if pos >= 0:
    value = orig_values[pos]
else:
    value = orig_values[0]  # 头部后向填充
```

#### 2.3.2 上采样·业务列：前向填充

与下采样相同算法（`bisect_right - 1`），因为下采样本质上也是"取历史最近"。

#### 2.3.3 上采样·标注列：最近邻（双向）

对每个网格时间点 `t`：
1. 若精确匹配原始时间戳 → 直接取值
2. 否则，比较前一个原始点 `prev` 和后一个原始点 `next` 到 `t` 的距离
3. **等距取前值**（`dist_prev <= dist_next` → 取前值）

**标注列禁止插值**——标注值是离散类别，不能做线性插值。

#### 2.3.4 网格对齐

原始间隔与目标间隔一致时：
1. 若网格时间点精确匹配原始时间戳 → 直接取值
2. 否则，找最近的原始点取值

### 2.4 偶发/区间缺失判定

重采样后，在标准网格上逐列扫描连续缺失段。

```
连续长度 ≥ K → 区间缺失
连续长度 < K → 偶发缺失（累加计数）
```

- K = 连续缺失判定阈值（默认 10，可通过元信息或 CLI 覆盖）
- 区间缺失记录：列名、起止索引、起止时间、长度
- 偶发缺失记录：每列偶发缺失总数

### 2.5 偶发缺失填充

#### 2.5.1 业务列：前向填充
- 从缺失位置向历史方向搜索最近非空值
- 遇到区间缺失边界 → 停止（不跨越）
- 头部无前值 → 后向填充

#### 2.5.2 标注列：最近邻
- 双向搜索最近非空值
- 不跨越区间缺失
- 等距取前值

#### 2.5.3 区间缺失
- **不填充**，保留空值

### 2.6 输出

- 重采样数据写入 `resampled_data.csv`（UTF-8-BOM）
- 重采样报告写入 `resampled_data_report.md`

## 4. 关键算法详解

### 4.1 口径B — 逐列单元格级别的对称绝对修改数据点数

核心思想：选择需要最少"数据搬动"的网格间隔。

- **逐列单元格级别**：对每一列独立计算其相对于网格需要新增和删除的数据点数，而非只在全局时间轴层面
- **对称**：删除和新增同等权重
- **绝对数**：统计修改的数据点数量，而非比例
- **平局选粗**：修改数据点数相同时，偏好更粗的网格（更大的间隔）

> ⚠️ **为什么不能只用全局时间轴集合差？**
> 合并后时间轴是所有源文件时间点的并集，高频列会"掩盖"低频列的真实采样率。
> 逐列计算是**精确的**：它准确统计了每个候选网格下所有列需要新增填充或删除丢弃的单元格总数。

示例（候选间隔来自各列众数间隔 {1s, 3s, 5s}）：

| 候选间隔 | 网格点数 | 需删除数据点 | 需新增数据点 | 总修改数据点数 | 结果 |
|---|---|---|---|---|---|
| 5秒 | 720 | 3776 | 530 | 4306 | **选中** |
| 3秒 | 1200 | 3295 | 1969 | 5264 | 落选 |
| 1秒 | 3600 | 0 | 8274 | 8274 | 落选 |

### 4.2 下采样 = 历史最近值

下采样时取输出时间点 `t` 之前最近的一个原始值。等价于**前向填充式的降采样**，符合因果性。

### 4.3 标注列最近邻 — 等距取前值

标注列（如 0/1 分类标签）的上采样使用双向最近邻而非前向填充。原因：标注值表示离散类别，前向填充可能导致标注边界向后偏移。等距时取前值符合因果直觉。

### 4.4 偶发缺失填充不跨越区间缺失

偶发缺失的填充搜索**不跨越区间缺失**——如果前向搜索遇到区间缺失的起始边界，立即停止。

原因：区间缺失表示传感器故障或通信中断，跨越它进行填充会产生无意义的虚构数据。

偶发缺失的填充范围仅限于"两个区间缺失之间"的连续有效数据段内。
