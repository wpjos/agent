# tsas-num-prep-resampler 元信息格式

> 本文档定义 resampler 接受的"元信息 MD"格式。AI 负责把用户的任意输入转写为此格式后传给脚本。

---

## 1. 总体约定

- 文件为 UTF-8 编码的 Markdown
- 二级标题（`##`）作为字段锚点，**标题文字必须完全匹配**
- 字段之间顺序不限
- 不识别的小节会被忽略，不会报错
- 列表项支持 `- col_name` 或 ``- `col_name` `` 两种形式

---

## 2. 字段一览

| 字段 | 是否必须 | 用途 | 缺失行为 |
|---|---|---|---|
| `## 列清单` | **必须** | 声明输入 CSV 的列名、角色、类型，让 resampler 知道哪些是业务列、哪些是标注列 | 报错中止 |
| `## 标注列` | 可选 | 显式列出标注列（覆盖 `## 列清单` 中"角色"列的自动识别） | 仅以 `## 列清单` 中"角色=标注"的列为标注列 |
| `## 目标网格间隔` | 可选 | 指定目标网格的秒数，跳过自动选择 | 自动按口径B从各列众数间隔中选择 |
| `## 区间缺失阈值` | 可选 | 指定区间缺失判定的 K 值 | 默认 10 |

> CLI 参数 `--grid-interval` 和 `--k` 优先级高于元信息文件中的对应字段。

---

## 3. 字段格式

### 3.1 `## 列清单`（必须）

二级 Markdown 表格，至少包含两列（列名、角色），可选第三列（类型）。

```markdown
## 列清单

| 列名 | 角色 | 类型 |
|---|---|---|
| time | 时间 | datetime |
| current_amps | 业务 | float |
| temperature_celsius | 业务 | float |
| vibration_rms | 业务 | float |
| label | 标注 | int |
```

- 表头必须以"列名"开头，分隔行（`|---|---|`）必须存在
- "角色"列识别关键字：包含"标注"或英文 `label` 即视为标注列
- "类型"列仅用于报告展示，不影响业务逻辑

### 3.2 `## 标注列`（可选）

显式列出标注列。出现时与 `## 列清单` 中角色识别的标注列做并集。

```markdown
## 标注列

- label
- anomaly_flag
```

### 3.3 `## 目标网格间隔`（可选）

单值字段，单位为秒。

```markdown
## 目标网格间隔

5
```

也支持带单位写法，会自动取首个数字：

```markdown
## 目标网格间隔

5 秒
```

### 3.4 `## 区间缺失阈值`（可选）

单值整数。

```markdown
## 区间缺失阈值

10
```

## 4. 完整示例

### 示例 1：最小必需

```markdown
# 重采样元信息

## 列清单

| 列名 | 角色 |
|---|---|
| time | 时间 |
| current_amps | 业务 |
| temperature_celsius | 业务 |
| vibration_rms | 业务 |
| label | 标注 |
```

### 示例 2：指定目标网格 + K

```markdown
# 重采样元信息

## 列清单

| 列名 | 角色 | 类型 |
|---|---|---|
| time | 时间 | datetime |
| current_amps | 业务 | float |
| temperature_celsius | 业务 | float |
| vibration_rms | 业务 | float |
| label | 标注 | int |

## 目标网格间隔

5

## 区间缺失阈值

15
```

### 示例 3：通过 `## 标注列` 显式声明

```markdown
# 重采样元信息

## 列清单

| 列名 | 角色 |
|---|---|
| time | 时间 |
| current_amps | 业务 |
| label_a | 业务 |
| label_b | 业务 |

## 标注列

- label_a
- label_b
```

---

## 5. 给 AI 的写作要点

- 当用户只描述"我有一份合并/清洗后的 CSV"时，请基于 CSV 的列名生成 `## 列清单`，并询问哪些是标注列
- 当用户说"想要 5 秒一个点"/"采样到 5 秒"时，写入 `## 目标网格间隔`
- 当用户说"超过 10 个点的空缺算长段缺失"时，写入 `## 区间缺失阈值`
- 不要在元信息中写算法实现细节——元信息只描述"输入是什么"和"用户偏好是什么"
