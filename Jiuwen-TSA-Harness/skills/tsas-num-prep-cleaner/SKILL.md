# tsas-num-prep-cleaner — 时间序列数据单元格治理

## 1. 职责

对一份已合并的时间序列 CSV 执行单元格级治理，输出可直接用于建模或下游处理的标准化 CSV 和治理报告：

- **显式缺失处理**：NA 标记字符串 → 统一空值
- **类型归一**：低格降级 / 高格升格，使每列类型一致
- **单位归一**：同物理量单位统一（需用户指定目标单位才执行）
- **类别编码**：类别列 → 整数（按用户映射或首次出现顺序）
- **标注编码**：标注列强制统一为 0/1
- **异常值/哨兵值检测**（可选）：超值域或匹配哨兵值的单元格置空

## 2. 输入要求

### 2.1 数据

一份合并后的 CSV 文件：
- 第一列为时间列（列名通常为 `time`），cleaner 不修改此列
- 其余列为待治理的业务列、标注列或类别列

### 2.2 元信息

一份 Markdown 格式的元信息文件，描述各列的数据类型、单位和角色。完整格式见 `references/metadata_spec.md`。

必须提供的元信息字段：

| 字段 | 用途 | 缺失行为 |
|---|---|---|
| 列清单 | 提供各列的数据类型、单位、角色 | 必须，缺失则报错 |

可选的元信息字段：

| 字段 | 用途 | 缺失行为 |
|---|---|---|
| 标注列 | 覆盖列清单中的角色字段 | 不覆盖 |
| 类别列 | 覆盖列清单中的角色字段 | 不覆盖 |
| 目标单位 | 指定列的目标单位，触发单位换算 | 不执行单位换算 |
| 类别映射 | 指定类别列的固定编码 | 按首次出现顺序编码 |
| 有效值域 | 指定列的有效范围 | 不做值域检查 |
| 哨兵值 | 指定列的哨兵值 | 不做哨兵值检查 |

### 2.3 元信息整理指南（给 AI）

用户提供的描述形式可能是自然语言、表格、JSON 等任意形式。在调用本 SKILL 前，AI 需将这些信息整理成 `references/metadata_spec.md` 定义的标准 MD 格式后传入 `--metadata` 参数。

整理时应：
- 必须提供列清单表（含所有非时间列）
- 标注列、类别列、目标单位、类别映射、有效值域、哨兵值按需提供

## 3. 输出

输出到 `--output-dir` 指定的目录（缺省为输入 CSV 同目录）：

| 文件 | 内容 |
|---|---|
| `cleaned_data.csv` | 治理后的数据 |
| `cleaned_data_report.md` | 治理报告（缺失处理、类型修改、单位转换、编码映射、错误警告） |

可通过 `--output` 和 `--report-output` 自定义文件名。

## 4. 用法

```bash
python <skill_dir>/scripts/cleaner.py \
    <merged_data.csv> \
    --metadata <metadata.md> \
    [--output-dir <dir>] \
    [--output <csv文件名>] \
    [--report-output <报告文件名>] \
    [--target-units '<json>'] \
    [--category-maps '<json>'] \
    [--label-cols <逗号分隔列名>] \
    [--enable-outlier-detection] \
    [--value-range '<json>'] \
    [--sentinels '<json>']
```

- `<skill_dir>` 为本 SKILL 所在目录
- `--metadata` / `-m` 必需，指向元信息 MD 文件
- `--target-units` 等 CLI 参数会覆盖元信息 MD 中的同名字段
- Python 环境：任一 Python 3.11+ 含 pandas/numpy（脚本启动时会自检依赖）

## 5. 处理流程

1. 解析元信息 MD，建立每列的角色、类型、单位
2. 显式缺失处理：NA 标记 → 空值
3. 类型归一（仅业务数据列）：降级或升格
4. 单位归一（仅业务数据列，需目标单位）
5. 类别编码（仅类别列）
6. 标注编码（仅标注列）：强制 0/1
7. 异常值/哨兵值检测（可选）
8. 写出 cleaned_data.csv 和 cleaned_data_report.md

详细算法见 `references/algorithm_spec.md`。

## 6. 报错条件

| 退出码 | 含义 |
|---|---|
| 0 | 成功 |
| 1 | 元信息错误（缺列清单、文件不存在、JSON 解析失败等） |
| 2 | 数据本身致命问题（标注列取值不合法等） |

常见报错：
- 元信息文件不存在 / 缺列清单 → 退出码 1
- `--target-units` 等 JSON 参数解析失败 → 退出码 1
- 标注列取值不在 {0,1} 也不在 {-1,1} → 退出码 2

## 7. 详细规范

- 元信息 MD 格式：`references/metadata_spec.md`
- 处理算法与报告格式：`references/algorithm_spec.md`

## 8. 注意事项

- 时间列不参与任何单元格治理
- 标注列、类别列跳过类型归一和单位归一，但参与显式缺失处理和各自编码
- 异常值/哨兵值默认关闭（异常检测任务中哨兵值可能是有效信息）
- 用户未指定目标单位时，不执行单位换算（仅记录原始单位到报告）
- 单位换算仅支持已知换算表中的单位；未知单位产生警告而非报错
