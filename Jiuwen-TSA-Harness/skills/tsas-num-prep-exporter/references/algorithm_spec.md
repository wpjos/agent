# tsas-num-prep-exporter 算法规格

> 本文档定义 exporter 的全部处理规则和算法逻辑，供 SKILL.md 引用。

---

## 1. 处理流程总览

```
读 CSV + 读元信息
     ↓
[1] 全空行切分（在所有业务列都为空的行处截断）
     ↓
[2] 列排序（time → 标注列 → 业务数据，按元信息顺序）
     ↓
[3] 最终质量校验（空值/时间等间隔/行列数）
     ↓
[4] 输出 CSV（单段或多段）+ 最终报告
```

---

## 2. 算法细节

### 2.1 全空行切分

#### 2.1.1 切分条件

**直接扫描数据**，找出全空行（一行中所有非时间列都为空值）作为切分点。

> 设计要点：
> 1. 不依赖任何外部报告，确保切分基于实际数据状态
> 2. 单列的区间缺失不触发切分——只有全空行（所有业务列+标注列都为空）才代表真正的数据断档

#### 2.1.2 切分算法

```
1. 扫描所有行，找出全空行的索引集合 cut_indices
2. 如果 cut_indices 为空 → 整体一个段，不切分
3. 否则：
   a. 将连续的全空行合并为一个"缺口"
   b. 在每个缺口处截断
   c. 缺口前的数据构成一个段，缺口后的数据构成下一个段
   d. 段内不含全空行
```

#### 2.1.3 输出文件命名

- 单段：`final_data.csv`
- 多段：`final_data_001.csv`、`final_data_002.csv`、...（3位序号）

#### 2.1.4 禁用切分

`--no-split` 选项跳过切分，保留为整段输出。报告中会注明"已禁用切分"。

### 2.2 列排序

统一列顺序为：

```
time → 标注列（按元信息列清单中出现顺序）→ 业务数据（按元信息列清单中出现顺序）
```

排序规则：
1. `time` 列始终在第一位
2. 标注列紧跟其后（从元信息 `## 列清单`「角色」识别 + `## 标注列` 显式声明）
3. 元信息中显式声明的剩余列按元信息中顺序排列
4. 未在元信息中出现的列，保持原顺序追加在最后

### 2.3 最终质量校验

| 检查项 | 级别 | 条件 |
|---|---|---|
| 空值检查 | warning | 任何业务列仍有空值 |
| 时间等间隔 | info | 时间轴只有 1 种间隔 |
| 时间不等间隔 | warning | 时间轴有多于 1 种间隔 |
| 行数不足 | error | 数据行数 < 2 |
| 行列数报告 | info | 最终行数和列数 |

校验在「合并所有段」后的完整数据上执行（多段时使用 `pd.concat`）。

### 2.4 输出

- 最终数据写入 `final_data.csv`（或多个 `final_data_NNN.csv`），UTF-8-BOM 编码
- 最终报告写入 `final_report.md`

## 3. 输出报告结构

报告章节是动态的：是否存在「全流程摘要」由元信息决定；是否存在「备注」也由元信息决定。

完整版（元信息包含全部可选字段）：

```markdown
# 最终预处理报告（tsas-num-prep-exporter）

## 1. 全流程摘要（仅当元信息提供 `## 全流程摘要` 时）
| 步骤 | SKILL | 关键结果 |

## 2. 输出段信息
- 段数、切分原因、各段起止时间和行数

## 3. 最终列清单
| 顺序 | 列名 | 角色 | 来源 |

## 4. 质量校验
| 级别 | 信息 |

## 5. 输出文件
- 文件列表

## 6. 错误/警告

## 7. 备注（仅当元信息提供 `## 备注` 时）
```

如果元信息未提供「全流程摘要」，则章节从「输出段信息」开始编号为 1。

## 4. 关键算法细节

### 4.1 全空行判定（直接扫描数据）

```python
def is_all_empty_row(row, business_cols):
    return all(is_empty(row[col]) for col in business_cols)
```

其中 `business_cols` 包含所有非时间列（标注列 + 业务数据列）。

> 单列区间缺失不切分的原因：
> 单列缺失只影响该列的数据连续性，其他列仍有有效值。
> 切分会破坏其他列的连续性，得不偿失。
> 只有全空行（所有列都缺失）才代表真正的数据断档。

### 4.2 连续缺口合并

```
cut_indices = {10, 11, 12, 50, 51}
→ 缺口1 = [10, 11, 12]（连续）
→ 缺口2 = [50, 51]（连续）
→ 段1 = [0, 9], 段2 = [13, 49], 段3 = [52, end]
```

### 4.3 列排序优先级

```
1. time                    （固定第一位）
2. 标注列                  （元信息中出现顺序）
3. 业务数据列              （元信息中出现顺序）
4. 未识别列                （原顺序追加）
```

### 4.4 NA 标记集合

```python
NA_TOKENS = {"", "na", "n/a", "#n/a", "nan", "null", "none", "-", "--", "nil"}
```

空字符串以及上述标记（大小写无关）都视为缺失。
