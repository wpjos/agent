# tsas-num-prep-exporter — 时间序列数据输出归一化

## 1. 职责

这是一个独立的时间序列数据输出归一化工具。给定**一份 CSV**（含时间列）和**一份元信息文件**，执行：

1. **全空行切分** — 在所有业务列都为空的行处截断，产出多个连续段
2. **列排序** — 统一列顺序为 `time → 标注列 → 业务数据`（按元信息中列清单顺序）
3. **最终质量校验** — 空值检查、时间等间隔检查、行数/列数检查

输出：
- `final_data.csv`（单段）或 `final_data_001.csv`、`final_data_002.csv`、...（多段）
- `final_report.md` — 输出归一化报告

## 2. 输入要求

### 2.1 数据文件

- CSV 文件，**首列为时间列**（列名 `time`），其余列为业务数据列或标注列
- 格式：UTF-8，第一行为列头
- 支持 CSV 中的缺失值表现为空字符串或常见 NA 标记

### 2.2 元信息文件（必选）

路径通过 `--metadata` 参数传入，Markdown 格式。详细格式规范见 `references/metadata_spec.md`。

**必须字段：**

| 字段 | 是否必须 | 用途 |
|---|---|---|
| `## 列清单` | **必须** | 声明 CSV 中各列的列名、角色（业务/标注/时间）、来源；**列在表格中的顺序决定最终输出的列顺序** |

**可选字段：**

| 字段 | 用途 | 缺失行为 |
|---|---|---|
| `## 标注列` | 显式列出标注列 | 仅以列清单中角色为"标注"的列为准 |
| `## 全流程摘要` | 在最终报告中追加 A→E 全流程摘要 | 报告跳过此小节 |
| `## 备注` | 在最终报告末尾追加备注段落 | 报告跳过此小节 |

## 3. 输出

| 输出项 | 文件名（可定制） | 说明 |
|---|---|---|
| 最终数据 | `final_data.csv` 或 `final_data_NNN.csv` | UTF-8-BOM 编码，列顺序统一 |
| 最终报告 | `final_report.md` | 包含段信息、列清单、质量校验、文件列表 |

退出码：
- **0**：成功
- **1**：IO/参数错误（文件不存在等）
- **2**：数据/元信息错误（缺少必需字段、缺少时间列等）

## 4. 用法

```bash
python exporter.py <input_csv> \
    --metadata <metadata.md> \
    [--output-dir <dir>] \
    [--output <csv文件名>] \
    [--report-output <报告文件名>] \
    [--no-split]
```

- `<input_csv>`：输入 CSV 文件路径（首列必须为 `time`）
- `--metadata` / `-m`：元信息文件路径（必需）
- `--output-dir` / `-o`：输出目录（缺省为输入文件所在目录）
- `--output`：CSV 输出文件名（缺省 `final_data.csv`；多段时自动加 `_NNN` 后缀）
- `--report-output`：报告输出文件名（缺省 `final_report.md`）
- `--no-split`：禁用全空行切分，强制单段输出

**Python 环境**：任一 Python 3.11+（含 pandas）。脚本启动时会自检依赖，缺失时打印友好提示。

## 5. 处理流程

### 5.1 解析元信息

从元信息 MD 中提取：
- 列清单（列名 → 角色、来源；**顺序敏感**）
- 标注列名列表
- 可选的全流程摘要、备注

### 5.2 全空行切分

直接扫描数据找全空行（所有非时间列都为空），在缺口处截断。
- 连续的多个全空行合并为一个"缺口"
- 单列的局部缺失不会触发切分
- `--no-split` 可禁用此环节

### 5.3 列排序

固定优先级：
1. `time` 在第一位
2. 标注列（按元信息中出现顺序）
3. 业务数据列（按元信息中出现顺序）
4. 未在元信息中声明的列保持原顺序追加

### 5.4 质量校验

对合并所有段后的完整数据执行：
- 空值检查（warning）
- 时间等间隔检查（info / warning）
- 行列数报告（info）

### 5.5 输出

- 单段 → `final_data.csv`
- 多段 → `final_data_001.csv`、`final_data_002.csv`、...
- 最终报告 → `final_report.md`

## 6. 报错条件

| 条件 | 退出码 | 提示 |
|---|---|---|
| 元信息文件不存在或不可读 | 1 | 检查路径 |
| CSV 文件不可读 | 1 | 检查路径 |
| 元信息缺少 `## 列清单` | 2 | 元信息须包含列清单 |
| CSV 中没有 `time` 列 | 2 | 数据文件首列须为时间 |
| 依赖缺失（pandas） | 1 | 换用已配置的 Python 环境 |
