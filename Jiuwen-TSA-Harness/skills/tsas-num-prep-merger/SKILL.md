# tsas-num-prep-merger — 时间序列数据合并对齐

## 1. 职责

merger 负责把分散在多个 CSV 文件中的时间序列数据"拼到一起"——执行单文件时间预处理（格式解析、精度恢复）、按数据分片形态选择合并策略（纵向拼接 / 横向外连接 / 混合）、统一时区和排序，最终输出一份合并后的 CSV 和 Markdown 报告。

合并后的数据是"宽表"格式：时间列为唯一索引，各业务列按来源对齐。时间点不对应产生的空值保留，不在本环节填充。

## 2. 输入要求

### 2.1 输入数据

一个或多个 CSV 文件，通过目录路径或单文件路径指定。

**数据必须满足**（不满足则报错中止）：
- 至少包含 1 个有效 CSV 文件
- 每个有效文件必须包含元信息中声明的时间列，且该列实际存在于文件中
- 时间列必须可解析为 datetime（timestamp 类型）或单调递增整数（sequence 类型）

**数据可以包含**（merger 自动处理，不报错）：
- 不同文件使用不同的时间列名（`time`、`timestamp`、`datetime` 等）→ 统一重命名为 `time`
- 时间精度缺失（如 `8:00` 重复 12 条表示 5 秒间隔）→ 自动精度恢复
- 不同文件包含不同时区 → 按优先级规则统一
- 业务列名带单位括号 → 自动去除括号、记录单位

### 2.2 元信息

merger 需要知道每个文件的时间列名、时间类型、分片形态。这些信息以 Markdown 文件形式提供，格式规范见 `references/metadata_spec.md`。

| 元信息项 | 必须/可选 | 用途 | 缺失时的行为 |
|---|---|---|---|
| 分片形态 | 必须 | 决定合并策略（A~F 对应不同合并方式） | 缺失 → 报错中止 |
| 各文件时间列名 | 必须 | 定位每个文件的时间列 | 缺失 → 报错中止 |
| 各文件时间类型 | 必须 | 区分时间戳还是整数序号（影响精度恢复逻辑） | 缺失 → 报错中止 |
| 各文件列清单 | 可选 | 用于报告中记录列来源 | 缺失 → 报告中列来源显示为"-" |
| 目标时区 | 可选 | 统一时间到指定时区 | 缺失 → 自动推断（有+08:00则统一+08:00 > 众数时区） |

### 2.3 元信息整理指南

- 如果用户提供了探查报告（无论由什么工具生成）→ AI 从中提取上述字段，整理成 merger 的元信息格式
- 如果用户口述"电流文件的时间列叫 time，温度文件的时间列叫 timestamp"→ AI 据此填写各文件时间列名
- 如果用户说"这几个文件按列分的"→ AI 将分片形态填为 C
- 如果用户没有任何分片形态认知，但能描述文件关系 → AI 根据描述判断（多个文件不同物理量按列拼→C，同结构文件按行拼→B）
- 如果用户完全不了解数据结构 → AI 应先建议做数据探查，拿到结构信息后再调用 merger

## 3. 输出

| 输出文件 | 内容 |
|---|---|
| `merged_data.csv` | 合并后的宽表。时间列统一为 `time`（naive datetime，不带时区），业务列按来源对齐，时间点不对应的空值保留 |
| `merged_data_report.md` | 合并报告：分片形态、合并策略、各文件预处理记录（精度恢复、时区）、列名归一记录、合并后列清单、时间范围、错误/警告 |

## 4. 用法

```bash
python <skill_dir>/scripts/merger.py <data_path> --metadata <path> [选项]
```

| 参数 | 必需 | 缺省 | 说明 |
|---|---|---|---|
| `data_path` | 是 | — | 原始数据文件或目录路径 |
| `--metadata` | 是 | — | 元信息文件路径（格式见 `references/metadata_spec.md`） |
| `--output-dir` / `-o` | 否 | `<data_path 同目录>` | 输出目录 |
| `--output` | 否 | `merged_data.csv` | 合并数据输出文件名 |
| `--report-output` | 否 | `merged_data_report.md` | 合并报告输出文件名 |
| `--timezone` | 否 | 自动推断 | 目标时区（如 `+08:00`），覆盖元信息和自动推断 |
| `--use-row-index` | 否 | 关闭 | 声明用行号作为序号（仅当文件无时间列时） |

- Python 环境：任一 Python 3.11+ 含 pandas/numpy（脚本启动时会自检依赖）

## 5. 处理流程

1. **读取元信息**：解析分片形态、各文件时间列名和类型、列清单。
2. **逐文件预处理**：读取 CSV，将时间列解析为 datetime；对 timestamp 类型做精度恢复（检测规律重复 → 按 60/N 左闭右开补秒位）；检测时区信息。
3. **列名归一**：时间列统一重命名为 `time`；非时间列去除单位括号。
4. **按分片形态合并**：A=单文件直通，B=纵向拼接，C=time 并集 outer join，D=组内列合并+组间行合并，E/F=outer join + 重叠区验证。
5. **全局时间治理**：统一时区（用户指定 > +08:00 > 众数），按时间升序排序。
6. **输出**：写入合并后的 CSV 和 Markdown 报告。

精度恢复算法、各形态合并策略、时区判定规则细节见 `references/algorithm_spec.md`。

## 6. 报错与警告条件

| 条件 | 级别 | 行为 |
|---|---|---|
| 元信息文件缺失 | **error** | 立即中止，退出码 1 |
| 元信息中无分片形态 | **error** | 立即中止，退出码 1 |
| 元信息中无文件信息 | **error** | 立即中止，退出码 1 |
| 文件的时间列不存在 | **error** | 记录到报告，退出码 2 |
| 精度恢复除不尽（即使展到微秒） | **error** | 记录到报告，退出码 2 |
| E/F 形态重叠区值不一致 | **error** | 记录到报告，退出码 2 |
| 同一时间戳恢复后时间点重叠 | **error** | 记录到报告，退出码 2 |
| 无有效文件可合并 | **error** | 记录到报告，退出码 2 |

**注意**：merger 的致命错误**不产出 CSV**（只产出报告），避免下游使用不完整数据。
