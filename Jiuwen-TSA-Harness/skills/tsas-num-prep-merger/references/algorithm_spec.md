# tsas-num-prep-merger 算法规格

> 本文档定义 merger 的全部处理规则和算法逻辑。

---

## 1. 逐文件读取与时间预处理

### 1.1 读取方式

- `dtype=str`，`keep_default_na=False`，`na_values=[]`（全列字符串读入，避免类型污染）
- 按文件名字典序处理
- 只处理元信息中声明的文件（元信息中未出现的文件被忽略）

### 1.2 时间列解析

- 从元信息获取时间列名
- timestamp 类型：尝试 `pd.to_datetime()`，失败则尝试预设格式列表
- sequence 类型：按整数序号处理，不强制解析为 datetime

### 1.3 精度恢复（仅 timestamp 类型）

**目的**：处理时间戳精度缺失——数据原始时间只到分钟（如 `8:00`），但实际采样间隔是秒级，同一分钟的时间戳被重复了多次。

**判定条件**：
- 检测时间列是否有规律重复（同一时间戳重复多次）
- 计算每个时间戳的重复次数
- 如果重复次数集中在众数附近（众数占比 > 50%）→ 判定为精度缺失

**恢复算法**：
```
对每个连续重复组（重复次数 N，时间戳 T）：
    interval = 60 / N
    for i in range(N):
        recovered_time[i] = T + timedelta(seconds=interval * i)
```
展开优先级：秒 → 毫秒 → 微秒（够用就停）：
- 先尝试 `60 / N`（秒级）
- 除不尽则 `60000 / N`（毫秒级）
- 仍除不尽则 `60000000 / N`（微秒级）
- 仍除不尽 → **报错中止**

**矛盾检查**：同一时间戳的不同重复组恢复后如果时间点重叠 → **报错中止**

### 1.4 列名归一

- 时间列统一重命名为 `time`（原始列名如 `timestamp`/`datetime` 会被替换）
- 非时间列去除单位括号（半角/全角），括号内单位单独记录

---

## 2. 时区处理

### 2.1 时区判定优先级

1. CLI 参数 `--timezone` 指定 → 转换到指定时区
2. 数据中含 `+08:00` 时区标记 → 统一到 `+08:00`
3. 以上都不满足 → 取众数时区（各文件中出现最多的时区）

### 2.2 输出格式

- 输出时间列**不带时区信息**（naive datetime）
- 时区转换过程在报告中说明

---

## 3. 分片形态合并策略

根据元信息中的分片形态，选择对应合并策略：

### A（单文件）
- 直接使用，无需合并

### B（按行划分）
- 纵向拼接所有文件（`pd.concat` axis=0）
- 列按归一名对齐（列顺序可能不同）
- 时间列统一为 `time`

### C（按列划分）
- time 列取所有文件的并集
- 对每个文件，按 time 列做 outer join
- 时间点不对应产生的空值保留（留给下游处理）

```python
all_times = sorted(union of all files' time columns)
for file in files:
    file = file.set_index('time').reindex(all_times)
merged = pd.concat([f for f in files], axis=1)
```

### D（对齐行列划分）
- 先按行分组（列名集合相同的文件为一组）
- 组内按 C 逻辑列合并
- 组间按 B 逻辑行合并

### E（不对齐行列划分）
- 对所有文件做 outer join（基于 time 列）
- **重叠区验证**（关键）：
  - 找出两个以上文件都覆盖的时间点
  - 对这些时间点的每个业务列，检查值是否字面相等
  - 任何不一致 → **报错中止**，详细说明冲突位置

```
对每对文件 (A, B)：
    overlap_times = A.time ∩ B.time
    对每个重叠时间点 t：
        对每个共有的列 c：
            if str(A[t, c]) != str(B[t, c]):
                报错：文件A和文件B在时间t的列c值不一致
```

### F（混合）
- 先分离对齐部分和不对齐部分
- 对齐部分按 D 处理
- 不对齐部分按 E 处理
- 合并结果

---

## 4. 分片形态判定参考

> 注意：分片形态由元信息提供，merger 不自行判定。以下是供 AI 整理元信息时参考的判定规则。

| 条件 | 形态 | 说明 |
|---|---|---|
| 只有 1 个文件 | A | 单文件，无需合并 |
| 多文件，列名集合完全相同 | B | 按行划分，纵向拼接 |
| 多文件，业务列不同，时间高度重叠 | C | 按列划分，横向外连接 |
| 多文件，既有行分组又有列分组，且对齐 | D | 组内列合并 + 组间行合并 |
| 多文件，既有行分组又有列分组，但不对齐 | E | 外连接 + 重叠区验证 |
| 部分对齐部分不对齐 | F | 混合处理 |
