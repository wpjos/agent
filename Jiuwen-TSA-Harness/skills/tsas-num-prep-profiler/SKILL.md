# tsas-num-prep-profiler — 时间序列数据探查识别

## 1. 职责

profiler 负责对时间序列原始 CSV 数据进行"摸底"——在不修改任何数据值的前提下，识别出数据的结构特征（列角色、类型、单位、分片形态），并产出一份结构化的探查报告。

这份报告记录了数据"长什么样"，供后续的数据治理决策使用。profiler **不输出任何数据文件**，只输出 Markdown 探查报告。

## 2. 输入要求

### 2.1 输入数据

一个或多个 CSV 文件，通过目录路径或单文件路径指定。

**数据必须满足**（不满足则报错中止）：
- 至少包含 1 个有效 CSV 文件（非空、非纯表头）
- 每个有效文件至少能识别出 1 个时间列（时间戳或整数序号）

**数据可以包含**（profiler 自动处理，不报错）：
- 空文件、纯表头文件 → 自动跳过并记录到报告
- NA 标记（`""`, `NA`, `N/A`, `#N/A`, `NaN`, `null`, `None`, `-`, `--`, `nil`，大小写不敏感）→ 统计为空值
- 表头下的单位行 → 自动识别并提取
- 哨兵值/异常值 → 当正常数据参与统计（不特殊处理）

### 2.2 元信息

profiler 主要是自动推断，但接受可选的用户提示。如果用户对某些列的角色有明确认知，可通过元信息文件提供，覆盖自动推断结果。

| 元信息项 | 必须/可选 | 用途 | 缺失时的行为 |
|---|---|---|---|
| 时间列名 | 可选 | 覆盖自动识别的时间列 | 缺失 → 自动推断（时间戳优先 > 整数序号） |
| 标注列名 | 可选 | 覆盖自动识别的标注列 | 缺失 → 自动识别（列名匹配 `label` + 取值 ⊆ {0,1} 或 {-1,1}） |
| 类别列名 | 可选 | 补充自动识别的类别列 | 缺失 → 自动识别（唯一值 ≤ 20 且占比 < 5%） |
| 跳过的文件 | 可选 | 指定不处理的文件名 | 缺失 → 处理所有 CSV 文件 |

元信息以 Markdown 文件形式提供，格式规范见 `references/metadata_spec.md`。AI 负责将用户提供的任意来源信息（自然语言描述、数据文档、口头说明等）整理成该格式。

### 2.3 元信息整理指南

- 如果用户说"`label` 那列是标注列" → 填入元信息的"标注列"小节
- 如果用户说"`timestamp` 那列才是时间" → 填入"时间列"小节
- 如果用户说"`device_id` 是类别" → 填入"类别列"小节
- 如果用户说"`bad.csv` 这个文件有问题别处理" → 填入"跳过的文件"小节
- 如果用户没有任何提示 → 不需要提供元信息文件，profiler 全自动推断

## 3. 输出

输出一份 Markdown 探查报告，包含以下小节：

| 小节 | 内容 |
|---|---|
| 概览 | 文件数量、分片形态判定、数据规模（总行数、总大小） |
| 各文件详情 | 每个文件的行数/列数/时间列/列清单表/单位信息 |
| 标注列 | 列名 + 编码方式（0/1 或 -1/1） |
| 类别列 | 列名 + 候选值列表 |
| 全量列汇总 | 所有非时间列的完整信息（列名/来源/类型/单位/时间范围/空值/唯一值/样例） |
| 跳过的文件 | 文件名 + 原因 |
| 错误/警告 | 致命错误和警告信息 |

## 4. 用法

```bash
python <skill_dir>/scripts/profiler.py <data_path> [选项]
```

| 参数 | 必需 | 缺省 | 说明 |
|---|---|---|---|
| `data_path` | 是 | — | 数据文件或目录路径 |
| `--metadata` | 否 | 无 | 元信息文件路径（包含用户提示，格式见 `references/metadata_spec.md`） |
| `--output` / `-o` | 否 | `<data_path 同目录>/raw_data_profiler.md` | 报告输出路径 |

- Python 环境：任一 Python 3.11+ 含 pandas/numpy（脚本启动时会自检依赖）

## 5. 处理流程

1. **文件发现**：读取 `data_path` 下所有 CSV 文件（递归），按文件名字典序排序。跳过元信息中指定的文件。
2. **单文件分析**：逐列判分（标注 > 复数 > 类别 > 数值 > 字符串），识别时间列（时间戳 > 整数序号），提取单位信息（列名括号 > 单位行 > 列名后缀）。元信息中指定的列角色覆盖自动识别。
3. **分片形态判定**：基于所有文件的归一化列名集合和时间重叠率，判定 A/B/C/D/E/F 六种形态。
4. **报告生成**：汇总所有分析结果，输出 Markdown 报告。

判定规则和阈值细节见 `references/algorithm_spec.md`。

## 6. 报错与警告条件

| 条件 | 级别 | 行为 |
|---|---|---|
| `data_path` 下无 CSV 文件 | **error** | 立即中止，退出码 1 |
| 文件内无时间列候选 | **error** | 记录到报告，退出码 2 |
| 文件内多个时间戳候选列 | **error** | 记录到报告，退出码 2 |
| 元信息指定的时间列不存在 | **error** | 记录到报告，退出码 2 |
| 空文件 / 纯表头 | info | 跳过并记录到报告 |
| CSV 读取失败 | info | 跳过并记录原因 |

## 7. 详细规范

- 元信息 MD 文件的字段定义和格式模板：`references/metadata_spec.md`
- 列类型判分规则、时间列识别逻辑、分片形态判定逻辑、单位提取规则：`references/algorithm_spec.md`
