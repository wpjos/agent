# tsas-num-prep-profiler 算法规格

> 本文档定义 profiler 的全部识别规则和判定逻辑。

---

## 1. 文件发现

- 若 `data_path` 是目录：读取目录下所有 `.csv` 文件（递归），按文件名字典序排序
- 若是单文件：只处理该文件
- 空文件（0 字节）或纯表头（只有表头行无数据行）：跳过并记录到报告
- 元信息中"跳过的文件"指定的文件：跳过并记录到报告

## 2. 单文件分析

### 2.1 读取方式

- 所有列值按**字符串**读入（`dtype=str`），避免 pandas 自动类型推断干扰判分
- NA 标记不转换为 NaN（`keep_default_na=False, na_values=[]`），保持原始字符串，由 profiler 自行判分

### 2.2 NA 标记集合

大小写不敏感，单独成值时识别为空值：
```
["", "NA", "N/A", "#N/A", "NaN", "null", "None", "-", "--", "nil"]
```

### 2.3 列类型判分顺序

对每个文件，逐列按以下顺序判定（命中即停止）：

```
标注 > 复数 > 类别(整数型) > 类别(字符串型) > 数值(整数) > 数值(浮点) > 字符串(描述)
```

### 2.4 判定规则

#### 标注列
- 元信息指定 → 直接标记
- 否则：列名匹配 `label`（大小写不敏感）**且** 非空值集合 ⊆ {0, 1} 或 {1, -1}
- 若多列都满足且用户未指定：取列名精确匹配 `label` 的那列，其余按类别处理

#### 复数列
- 剔除 NA 标记后，可解析为 `"a+bj"` 或 `"a+bi"` 格式的值占比 ≥ 90%

#### 类别列（整数型）
- 剔除 NA 标记后：
  - 所有值可解析为整数
  - 唯一值数 ≤ 20
  - 唯一值数 / 总行数 < 5%

#### 类别列（字符串型）
- 剔除 NA 标记后：
  - 列为字符串类型
  - 唯一值数 / 总行数 < 5%

#### 数值列（整数）
- 剔除 NA 标记后，所有值可解析为整数（且不满足类别条件）

#### 数值列（浮点）
- 剔除 NA 标记后，可解析为数值（含小数/科学计数）的值占比 ≥ 90%

#### 字符串列
- 不满足以上任何条件的列

### 2.5 哨兵值

除非用户明确指定有效值域或哨兵值，否则**不特殊处理**，当正常数据参与统计。

## 3. 时间列识别

### 3.1 时间戳列判定
- 尝试将列值解析为 datetime（先尝试 pandas 智能解析，再逐一尝试预设格式）
- 采样前 100 个非空值，成功率 ≥ 90% → 判定为时间戳列
- 判定后解析全列获取 min/max

### 3.2 整数序号列判定
优先级（组合判据最强）：
1. **[取值单调递增] 且 [唯一值占比高（> 80%）]** → 最强信号
2. 列名提示（time/date/timestamp/datetime/index/no/sequence_no 等，大小写不敏感）
3. 位于第一列

满足条件 1 即判定；若不满足 1 但满足 2+3，也判定。

### 3.3 元信息覆盖

元信息中指定的时间列名 → 直接标记为时间列，不做自动识别。

### 3.4 多时间戳候选列
- 一个文件内出现多个时间戳列 → **报错（致命）**

### 3.5 无时间列
- 文件中无任何时间列候选 → **报错（致命）**，提示用户可声明用行号

## 4. 单位信息提取

按以下优先级提取（命中即停止）：

1. **列名括号**：匹配 `xxx(单位)` 或 `xxx（单位）`（半角/全角），提取括号内容
2. **单独数据行**：表头下紧接的 1~3 行，若 ≥70% 的单元格为非数字 → 判定为单位行
3. **列名后缀**：列名末尾匹配已知单位（按长度降序匹配，避免 "a" 误匹配 "amps"）

### 已知单位清单

电学：A/amps/amp/ma/ua/v/mv/w/kw/mw/va/kva/ah/mah/hz/khz/mhz/ohm/f/uf/pf/h
力学/运动：n/kn/pa/kpa/mpa/gpa/bar/mbar/m·s/m·s²/km·h/rpm/g/mg/kg/t/mm/cm/m/km/um/nm
温度：celsius/fahrenheit/°c/℃/°f/℉/k
时间：s/ms/us/ns/min/h/d
其他：%/ppm/ppb/db/lux/ph/l/ml/mol/mmol/atm/rms

## 5. 分片形态判定

基于所有有效文件的分析结果，判定分片形态：

| 条件 | 判定 |
|---|---|
| 只有 1 个有效文件 | A |
| 多文件，归一化列名集合完全相同 | B（按行划分） |
| 多文件，业务列有交集，时间高度重叠（≥50%） | C（按列划分） |
| 多文件，业务列有交集，存在重复列集合 | D（行列对齐） |
| 多文件，业务列无交集或时间不重叠 | E（行列不对齐） |

### 归一化规则

- 时间列统一映射为 `time`
- 其余列去除单位括号后的列名

### 时间重叠率计算

取所有文件时间范围的交集时长 / 最短文件时长。≥50% 视为"高度重叠"。

```markdown
- 分片形态判定：X（原因）

### 文件：xxx.csv
- 行数 / 列数 / 文件大小
- 时间列：xxx（类型：时间戳/整数序号）
- 时间格式样例
- 时间范围：min ~ max
- 列清单表（列名/数据类型/空值数量/非空数量/唯一值数/单位/样例值）
- 单位信息

## 3. 标注列
| 文件 | 列名 | 编码方式 |

## 4. 类别列
| 文件 | 列名 | 类型 | 候选值 |

## 5. 全量列汇总
| 列名 | 来源文件 | 数据类型 | 单位 | 时间范围 | 非空数量 | 空值数量 | 唯一值数 | 样例值 |

## 6. 跳过的文件
| 文件名 | 原因 |

## 7. 错误/警告
```
