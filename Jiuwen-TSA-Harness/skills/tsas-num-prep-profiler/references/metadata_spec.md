# tsas-num-prep-profiler 元信息格式规范

> 本文档定义 profiler 接受的元信息 MD 文件的字段和格式。
> AI 负责将用户提供的任意来源信息整理成此格式后传给脚本。

---

## 1. 元信息文件是可选的

profiler 主要是自动推断。只有当用户对某些列的角色有明确认知、需要覆盖自动推断时，才需要提供元信息文件。如果用户没有任何提示，不需要提供。

---

## 2. 元信息字段

| 字段 | 必须/可选 | 说明 |
|---|---|---|
| 时间列名 | 可选 | 指定哪一列是时间列，覆盖自动识别 |
| 标注列名 | 可选 | 指定哪些列是标注列，覆盖自动识别 |
| 类别列名 | 可选 | 指定哪些列是类别列，补充自动识别 |
| 跳过的文件 | 可选 | 指定不处理的文件名 |

---

## 3. 标准 MD 格式模板

元信息文件使用以下 Markdown 格式。未提供的小节可省略，profiler 会对应字段走自动推断。

```markdown
# 元信息

## 时间列
<列名>

## 标注列
<列名1>, <列名2>

## 类别列
<列名1>, <列名2>

## 跳过的文件
<文件名1>, <文件名2>
```

### 字段说明

- **时间列**：单个列名。指定后 profiler 直接用此列作为时间列，不做自动识别。
- **标注列**：逗号分隔的列名列表。指定后这些列被强制标记为标注列。
- **类别列**：逗号分隔的列名列表。指定后这些列被强制标记为类别列。
- **跳过的文件**：逗号分隔的文件名列表。指定的文件被跳过（不读取、不分析）。

### 格式细节

- 每个小节的标题行（`## xxx`）是 profiler 的解析锚点，必须精确匹配
- 列名/文件名用逗号分隔，逗号后可有可无空格
- 列名不需要反引号包围（写了也会被去除）
- 空行忽略
- 只有列名/文件名所在的行（小节标题后的第一个非空行）会被解析

---

## 4. 示例

### 示例 1：用户指定时间列和标注列

```markdown
# 元信息

## 时间列
timestamp

## 标注列
label
```

### 示例 2：用户指定跳过某文件

```markdown
# 元信息

## 跳过的文件
bad_data.csv, incomplete.csv
```

### 示例 3：用户指定多个类别列

```markdown
# 元信息

## 类别列
device_type, fault_code, location
```
